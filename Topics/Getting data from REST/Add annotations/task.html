This problem is fully supported by version 2021.9 of the EduTools plugin and newer. To avoid possible errors, please make sure that the plugin is updated. Alternatively, you can solve it using the web interface.

Analyze the code and transform the plain Java class with one method to Spring controller by adding necessary annotations.

The program should have the following endpoint:

GET /secret_numbers/{id}: returns a secret number with a specified id as JSON object with one field secret_number. If a secret number with a specified id is not present, the value of secret_number should be equal to -1.

Do not change the content of the map or the logic of the method. Just add annotations needed for this task.

Example 1: Response for a GET /secret_numbers/4 request:

{
   "secret_number": 383832238472
}

Example 2: Response for a GET /secret_numbers/100 request:

{
   "secret_number": -1
}
